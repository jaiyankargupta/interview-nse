**Task:**

2. Fix the algorithm
3. Test thoroughly to ensure accuracy
4. Verify all edge cases
All while maintaining the rest of the application's functionality."

**Action:**

**Hour 1-4 - Diagnosis:**
- Recreated the bug with test data
- Traced through the calculation logic
- Discovered I was using the last purchase price instead of weighted average
- Understood the correct formula: Avg Cost = Total Amount Paid / Total Quantity

**Hour 5-12 - Implementation:**
```javascript
// Old (Wrong) approach
const profitLoss = (currentPrice - lastPurchasePrice) * totalQuantity;

// New (Correct) approach
const totalAmountPaid = holdings.reduce((sum, h) => 
    sum + (h.quantity * h.purchasePrice), 0);
const totalShares = holdings.reduce((sum, h) => sum + h.quantity, 0);
const avgCost = totalAmountPaid / totalShares;
const profitLoss = (currentPrice - avgCost) * totalShares;
```
- Refactored calculation logic
- Updated database schema to store necessary data
- Tested with complex scenarios

**Hour 13-24 - Testing:**
- Created test cases:
  - Single purchase
  - Multiple purchases at same price
  - Multiple purchases at different prices
  - Sells followed by buys
  - Edge cases (zero quantity, very large numbers)
- Manual verification against Excel calculations
- Fixed additional bugs found during testing

**Hour 25-48 - Final Verification:**
- Tested entire application end-to-end
- Verified UI displayed correct calculations
- Checked historical data migration
- Got a friend to review my math
- Deployed with confidence

**Result:**
"I fixed the bug, improved the calculation accuracy, and actually made the feature better than originally planned. The application now correctly handles complex trading scenarios. I met my deadline with a more robust solution."

**What I Learned:**

## Mock Interview Practice

**Practice Routine:**

**1. Record Yourself:**
- Use phone/laptop camera
- Answer common questions out loud
- Watch playback and critique yourself

**2. Time Your Answers:**
- Most answers: 1-2 minutes
- "Tell me about yourself": 2 minutes max
- Don't ramble beyond 3 minutes

**3. Practice with Others:**
- Ask a friend/family member to play interviewer
- Do mock video calls for remote interview practice
- Get feedback on:
  - Eye contact
  - Clarity of speech
  - Confidence level
  - Body language

**4. Prepare Answer Frameworks:**
- Use STAR method for behavioral questions:
  - **S**ituation: Context
  - **T**ask: Your responsibility
  - **A**ction: What you did
  - **R**esult: Outcome and learning

**5. Anticipate Follow-Ups:**
For each answer, think: "What might they ask next?"
Example: If you say "I'm a fast learner," expect: "Can you give an example?"
